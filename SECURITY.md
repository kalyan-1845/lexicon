# Security Policy

## Supported Versions

We currently provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.3.x   | :white_check_mark: |
| 1.2.x   | :x:                |
| < 1.1   | :x:                |

## Reporting a Vulnerability

We take the security of Lexicon AI seriously. If you believe you have found a security vulnerability, please report it to us by following these steps:

1. **Do not open a public issue.** This could expose users to risk before a fix is ready.
2. Email your findings to **security@lexicon.ai** (simulated for NSoC'26).
3. Provide a detailed description of the vulnerability, including steps to reproduce.
4. We will acknowledge your report within 48 hours and provide a timeline for a fix.

### Scope

The following areas are in scope for security reports:
- API Authentication & Authorization
- Data Isolation (Workspace/Collection leakage)
- Cross-Site Scripting (XSS) in Markdown rendering
- Remote Code Execution (RCE) in Agent tools

Thank you for helping keep Lexicon AI secure for all researchers!
